Git常用命令

Posted on 2018-04-28

主要步骤

添加(add), 提交(commit), 远程推送(push), 远程克隆(clone), 远程更新(pull),

其它命令: branch, tag, remote, checkout, merge, log, status, fetch, rebase等

命令

git branch 查看本地分支
git branch -a 查看远程所有分支
git branch 新增本地分支
git checkout 切换到本地分支
git branch -d 删除分支(注意必须先切换到其它分支才行)
git branch -av 显示提交信息
git branch -avv 显示本地分支和那个远程分支关联

git push origin dev -u 将本地修改提交到远程的 dev 分支(如果远程不存在dev分支, 需要加 -u)

注意

  1. 将本地的文件 commit 到一个分支后, 再其它分支是无法看见该文件的, 除非 合并分支
    例如:
    有两个分支: master, dev
    有一个文件 a.txt, 将它提交到 dev 分支, 在 master 分支就无法看见该文件

    此时在 master 分支里面无法看见 a.txt 这个文件

  2. 解决冲突
    例如:
    当你同时修改 a.txt时, 使用 git pull, 就会显示 dev | merging
    现在需要手动编辑该文件, 然后再次提交

  3. 一种常见的情况
    你在远程新建一个 master 分支, 并且给里面新建了一个 readme.md 文件
    你在本地又新建了一个 master 分支, 并且新建了 a.txt
    然后你 git remote -a origin http://github.com/text.git
    现在就回出现 即不能提交又不能更新的问题. how?
    需要将本地 master 分支与远程 master 分支进行关联
